# 基于SpringBoot+Vue3＋微信小程序的佛山本地智能气象出行服务系统设计与实现的需求分析

## 一、项目概述

### 1.1 项目背景

佛山地处亚热带季风气候区，夏季高温多雨、汛期暴雨集中，台风、强对流、高温等气象灾害频发，市民日常通勤、休闲出行常受突发天气影响。传统气象工具信息泛化，缺少与本地出行场景的深度结合，无法针对佛山易积水路段、通勤路线、本地景点给出场景化气象指引。

本项目以微信小程序为用户入口、Vue3 搭建管理后台、SpringBoot 提供后端服务，聚焦佛山本地场景，将精准气象数据与出行建议深度融合，为市民提供轻量化、智能化的出行气象服务。

### 1.2 项目目标

1. 为佛山用户提供精确到镇街的实时气象、多时段预报与灾害预警信息；
2. 结合气象数据与地图能力，输出差异化智能出行建议与风险提示；
3. 搭建 Web 管理后台，实现气象数据运维、出行内容管理、用户反馈处理；
4. 覆盖日常通勤、休闲出游、应急避险等多场景，降低天气对市民出行的负面影响。

### 1.3 项目范围

- 覆盖区域：佛山市禅城、南海、顺德、高明、三水五区及下属镇街；
- 用户终端：微信小程序（C 端用户）、PC Web 管理后台（运营管理员）；
- 核心能力：气象查询、智能出行推荐、预警推送、后台内容运维；
- 边界说明：以气象辅助出行为核心，不包含实时公交调度、网约车派单等深度出行服务，导航能力依托第三方地图 SDK 实现。

## 二、用户角色分析

### 2.1 小程序端普通用户

佛山本地居民、通勤族、短途游客，核心诉求是快速查询本地天气、获取出行指引、接收灾害预警。典型场景包括：上下班通勤查看天气、出游前查询目的地天气、暴雨天查看积水路段提醒。

### 2.2 Web 管理端系统管理员

负责系统日常运维，分为数据运维岗、内容运营岗、超级管理员三类，核心工作包括气象数据同步校验、预警发布、出行地点维护、用户反馈处理、系统权限配置。

## 三、功能性需求分析

### 3.1 微信小程序端（用户端）

#### 3.1.1 气象信息服务模块

- **实时天气展示**：自动定位用户所在佛山区域（精确到镇街），展示当前温度、体感温度、湿度、风向风力、气压、能见度；支持手动切换五区各镇街点位。
- **多时段预报**：提供未来 24 小时逐小时预报、7 天详细预报、15 天天气趋势，包含温度、降水概率、风力风向变化曲线。
- **气象预警中心**：接入暴雨、台风、高温、寒冷、雷电、大风等佛山高频预警类型，展示预警级别、发布时间、影响范围、官方防御指引。
- **生活指数服务**：展示 AQI 空气质量与污染物数据；提供穿衣、紫外线、洗车、运动、晾晒、旅游等生活指数，阈值适配佛山本地气候特征。
- **气象可视化**：以折线图、柱状图展示温度、降水趋势，支持按日 / 按小时切换查看。

#### 3.1.2 智能出行服务模块

- **出行天气查询**：用户输入出发地、目的地与出行时间，系统生成对应时段的出行天气报告，包含沿途天气变化、降水风险。

- 场景化出行建议

  ：基于天气状况输出差异化指引：

  - 降雨 / 暴雨：提醒携带雨具，标注佛山易积水路段，优先推荐地铁、公交等遮蔽性出行方式；
  - 高温暴晒：提醒防晒补水，推荐阴凉路线与室内中转点；
  - 台风 / 强对流：建议减少外出，标注风险路段与就近避险场所；
  - 晴好天气：推荐步行、骑行等绿色出行方式。

- **通勤出行定制**：用户可设置家、公司等常用地点与通勤时段，系统每日定时推送通勤路线天气与出行提醒，适配广佛跨城通勤场景。

- **本地出行推荐**：结合当日天气推荐适配的佛山目的地：晴好天推荐千灯湖、西樵山、顺峰山等户外景点；降雨天推荐岭南天地、祖庙博物馆、科技馆等室内场所。

- **地图能力集成**：接入第三方地图小程序 SDK，展示地点位置与路线示意，支持一键跳转原生地图导航。

#### 3.1.3 消息提醒模块

- **订阅消息推送**：用户授权后，气象预警、通勤提醒、恶劣天气提示通过微信服务通知实时触达。
- **提醒自定义**：用户可自主开关预警提醒、通勤提醒，设置提醒级别阈值（如仅接收橙色及以上预警）。

#### 3.1.4 个人中心模块

- 常用地点管理：新增、编辑、删除家、公司等收藏地址；
- 系统设置：通知开关、定位权限管理、隐私设置；
- 意见反馈：提交功能建议、气象纠错，支持上传图片凭证；
- 关于系统：版本信息、使用帮助、免责声明。

### 3.2 Web 管理后台端（Vue3）

#### 3.2.1 数据概览模块

- 系统数据大盘：展示总用户数、日活、访问量、预警发布数、反馈处理率等核心指标；
- 气象数据统计：佛山近期气温、降水、预警次数可视化图表，支持按区域筛选；
- 出行服务统计：出行查询次数、热门目的地、通勤用户占比等运营数据。

#### 3.2.2 气象数据管理模块

- 实时数据管理：支持自动 / 手动同步佛山各区域实时气象数据，可人工修正异常数据；
- 预报数据管理：维护逐小时、7 天、15 天预报数据，支持手动调整与发布；
- 预警信息管理：新增、编辑、发布、下架预警内容，配置预警级别、影响区域、防御指引，支持一键推送至小程序；
- 生活指数配置：自定义各类生活指数的计算规则、展示文案与阈值；
- 区域信息维护：管理佛山五区及下属镇街的行政编码、地理坐标。

#### 3.2.3 出行服务管理模块

- 出行地点管理：维护佛山景点、商圈、交通枢纽等地点信息，配置适配天气标签、推荐指数、封面图；
- 风险路段管理：录入易积水点、易堵点、施工路段等信息，关联天气触发条件（如暴雨天气自动展示积水点）；
- 出行规则配置：配置智能推荐引擎规则，定义不同降水概率、温度、风力对应的出行建议文案；
- 通勤服务配置：设置通勤提醒时间规则、推送消息模板。

#### 3.2.4 用户与反馈管理模块

- 用户列表：查看小程序注册用户信息，支持按区域、注册时间筛选；
- 反馈处理：受理用户意见反馈，标记处理状态，在线回复用户。

#### 3.2.5 系统管理模块

- 管理员账号与角色权限：基于 RBAC 实现多级权限分配；
- 操作日志：记录所有后台操作，支持审计追溯；
- 系统参数：配置气象同步频率、接口密钥、推送开关等全局参数。

### 3.3 后端核心服务（SpringBoot）

#### 3.3.1 第三方数据对接服务

- 气象数据对接：对接专业气象 API，定时拉取佛山区域实时天气、预报、预警、空气质量数据，完成清洗、校验与存储；
- 地图服务对接：集成地理编码、路线查询、地址解析能力；
- 微信接口对接：实现小程序登录、用户信息获取、订阅消息推送。

#### 3.3.2 智能出行推荐引擎

- 天气因子提取：解析温度、降水概率、风力、预警等级等核心气象指标；
- 规则匹配计算：基于规则库匹配对应出行建议、风险提示与目的地推荐；
- 用户偏好适配：结合用户收藏地点、出行习惯优化推荐结果。

#### 3.3.3 消息推送服务

- 订阅消息管理：维护微信模板与用户订阅关系；
- 定时推送：基于定时任务实现通勤提醒、每日天气日报推送；
- 预警即时推送：预警发布后触发实时推送任务。

#### 3.3.4 权限与安全服务

- 小程序用户认证：基于微信 openid 实现身份与会话管理；
- 后台认证鉴权：JWT 登录认证 + 接口权限校验；
- 接口安全：签名校验、防重放、SQL 注入防护、限流控制。

## 四、非功能性需求分析

### 4.1 性能需求

- 小程序首屏加载≤2 秒，气象查询响应≤1 秒，出行建议生成≤1.5 秒；
- 管理后台页面加载≤3 秒，数据查询响应≤2 秒；
- 支持 1000 + 用户并发访问，核心接口 QPS≥500；
- 数据更新：实时天气每 30 分钟同步，预警数据实时同步，日预报每日更新 4 次。

### 4.2 可靠性需求

- 系统年可用率≥99.5%，月度故障时长≤3.6 小时；
- 气象数据准确率≥98%，预警推送延迟≤5 分钟；
- 数据库每日自动备份，支持数据恢复；第三方接口异常时降级展示缓存数据，不引发系统崩溃。

### 4.3 易用性需求

- 小程序核心功能 3 步内可达，操作符合微信用户习惯；
- 后台采用模块化布局，提供操作提示，降低运维学习成本。

### 4.4 安全性需求

- 用户定位、个人信息严格保密，符合《个人信息保护法》要求；
- 敏感信息加密存储，接口统一鉴权，防止非法调用与数据爬取；
- 后台支持登录审计、操作日志追溯，防止越权操作。

### 4.5 兼容性需求

- 小程序兼容微信 8.0 及以上版本，适配 iOS/Android 主流机型；
- 管理后台兼容 Chrome、Edge、Firefox 最新两个正式版本；
- 后台适配 1920*1080 及以上主流分辨率。

### 4.6 可扩展性需求

- 功能扩展：预留积水实况、公交天气联动、同城活动等接口；
- 区域扩展：架构支持后续拓展至珠三角其他城市；
- 性能扩展：支持集群部署，可通过增加节点提升承载能力。

## 五、核心业务流程

### 5.1 用户出行查询流程

1. 用户打开小程序，授权定位获取佛山当前位置；
2. 后端返回实时天气与预报数据，前端渲染展示；
3. 用户输入出发地、目的地与出行时间提交查询；
4. 后端调用地图接口获取路线，同时匹配对应时段气象数据；
5. 智能推荐引擎生成出行建议与风险提示；
6. 结果返回小程序，向用户展示完整出行天气报告。

### 5.2 气象预警发布推送流程

1. 第三方 API 推送预警数据，或管理员手动录入预警信息；
2. 后台校验预警级别与影响区域，生成正式预警内容；
3. 推送服务筛选订阅该区域预警的用户；
4. 通过微信订阅消息向用户推送通知；
5. 小程序预警中心同步更新预警详情与防御指引。

### 5.3 后台管理操作流程

1. 管理员输入账号密码登录后台；
2. 系统验证身份与权限，返回对应菜单；
3. 管理员执行数据同步、预警发布、内容编辑等操作；
4. 系统记录操作日志，更新数据库；
5. 数据变更同步至小程序端生效。

## 六、数据需求分析

### 6.1 核心数据实体

- 用户信息：用户 ID、openid、昵称、头像、定位区域、注册时间；
- 区域信息：区域 ID、名称、行政编码、上级区域、经纬度；
- 实时气象数据：温度、体感温度、湿度、风向风力、更新时间、关联区域；
- 预报气象数据：预报日期、最高 / 最低温、天气状况、降水概率、关联区域；
- 气象预警数据：预警类型、级别、内容、发布 / 失效时间、影响区域、状态；
- 出行地点数据：名称、地址、经纬度、适配天气标签、推荐等级、类型；
- 风险路段数据：路段名称、位置、风险类型、触发天气条件、备注；
- 用户收藏地点：用户 ID、地点名称、地址、标签（家 / 公司）；
- 意见反馈：用户 ID、内容、图片、提交时间、处理状态、回复；
- 管理员与日志：账号、角色、操作记录、操作时间、IP。

### 6.2 数据来源

- 气象数据：第三方商业气象 API + 佛山气象公开数据；
- 地理地图数据：腾讯 / 高德地图开放平台；
- 用户数据：用户授权获取与自主提交；
- 运营数据：后台管理员录入维护。

## 七、技术选型说明

### 7.1 后端（SpringBoot 生态）

- 核心框架：Spring Boot 3.x
- 持久层：MyBatis-Plus
- 数据库：MySQL 8.0
- 缓存：Redis 7.x（缓存热点气象数据、用户会话）
- 定时任务：Quartz
- 认证：JWT + Spring Security
- 接口文档：Knife4j
- 部署：Linux + Nginx + Docker

### 7.2 管理后台（Vue3 生态）

- 核心框架：Vue 3 + Vite
- UI 组件：Element Plus
- 状态管理：Pinia
- 路由：Vue Router 4
- 图表：ECharts
- 请求工具：Axios

### 7.3 微信小程序端

- 开发框架：微信小程序原生
- UI 组件：Vant Weapp
- 地图：腾讯地图小程序 SDK
- 图表：uCharts

### 7.4 第三方服务

- 气象数据：和风天气 / 心知天气开放平台
- 地图服务：腾讯位置服务
- 消息推送：微信小程序订阅消息
